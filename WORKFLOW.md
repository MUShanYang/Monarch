## 关键数据流

### 输入
- `story/story_bible.md` - 故事设定
- `story/style_guide.md` - 风格指南
- `story/volume_outline.md` - 卷大纲
- `story/chapter_summaries.md` - 章节摘要
- `story/subplot_board.md` - 副线板
- `story/emotional_arcs.md` - 情感弧线
- `story/character_matrix.md` - 角色矩阵
- `story/db/entities_db.json` - 实体数据库
- `story/db/narrative_ledger.json` - 叙事账本
- `story/db/motif_index.json` - 母题索引

### 输出
- `chapters/NNNN_<title>.md` - 章节文本
- `story/runtime/chapter-NNNN.state-diff.json` - 状态差异
- `story/runtime/chapter-NNNN.trace.json` - 执行追踪
- 更新所有真相文件

## 性能指标

### 时间消耗 (第21章示例)
- 初始化: ~2s
- 节拍 1 (environment): ~7分钟
  - 3×3 speculative 生成: ~6分钟
  - 审计: ~1分钟
- 节拍 2 (interiority): ~8分钟
- 节拍 3 (action): ~3分钟
- 总计: ~18分钟

### Token 消耗
- 每个候选生成: ~1000-1500 input tokens
- 每个节拍 9 个候选: ~9000-13500 input tokens
- 审计: ~500 tokens per candidate
- 总计每节拍: ~15000-20000 tokens

### 重试机制
- LLM 返回空响应时自动重试
- 第1次: 移除 stop sequences
- 第2次: 扩展 token budget
- 最多重试 2 次

## 问题诊断

### 常见警告
1. **LLM 流式响应无文本内容**
   - 原因: stop sequences 过早触发
   - 解决: 自动重试，移除 stop sequences

2. **字数归一化**
   - 原因: 生成字数超出目标范围
   - 解决: 自动裁剪或扩展

3. **审计未通过**
   - 原因: 违反 HardBan、专有名词错误等
   - 解决: 降级使用得分最高的候选

4. **场景切换突兀**
   - 原因: 缺乏过渡机制
   - 解决: 人工复核或重写

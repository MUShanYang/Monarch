# 角色知识追踪系统使用指南

## 问题背景

AI 生成小说时常见的严重问题：**角色说出他们不应该知道的信息**

例如：
- 角色 A 在密室中发现了秘密，但角色 B 在下一章就知道了这个秘密（没有任何信息传递）
- 角色只是怀疑某事，但 AI 让他用确定的语气说出来
- 角色提及了其他角色的私密想法或秘密

这会导致**剧情逻辑崩溃**，严重影响小说质量。

## 解决方案

Monarch 的**知识追踪系统**通过以下方式解决这个问题：

1. **追踪每个角色的知识状态**：记录每个角色在每章知道什么、不知道什么
2. **自动验证生成内容**：每个 beat 生成后自动检查是否违反知识边界
3. **自动提取新知识**：从对话中提取"知道"、"发现"等信息，更新知识状态
4. **提供修复建议**：发现违规时给出具体的修复建议

## 系统架构

```
KnowledgeTracker (知识追踪器)
  ├─ 追踪每个角色的知识状态
  ├─ 保存到 story/character_knowledge.md
  └─ 提供验证接口

KnowledgeValidator (知识验证器)
  ├─ 验证 beat 是否违反知识边界
  ├─ 自动提取新知识
  └─ 决定是否重试生成

ChapterPipelineAdapter
  ├─ 初始化时加载知识状态
  ├─ 每个 beat 生成后验证
  └─ 章节完成后保存知识状态
```

## 文件格式：character_knowledge.md

系统会在 `story/character_knowledge.md` 中追踪角色知识：

```markdown
# 角色知识追踪

此文件追踪每个角色在故事中知道什么、不知道什么。
用于防止 AI 让角色说出他们不应该知道的信息。

## 张三 (zhang-san)

**确定知道** (5 项):
- 李四是卧底 (第 3 章)
- 宝藏在北山 (第 5 章)
- 师父已经去世 (第 1 章)
- 王五背叛了组织 (第 7 章)
- 密码是"天地玄黄" (第 8 章)

**怀疑** (2 项):
- 赵六可能是幕后黑手 (第 6 章)
- 宝藏可能是陷阱 (第 7 章)

**明确不知道** (3 项):
- 李四的真实身份
- 师父留下的遗言内容
- 组织的真正目的

## 李四 (li-si)

**确定知道** (3 项):
- 自己是卧底 (第 1 章)
- 张三在寻找宝藏 (第 4 章)
- 组织的真正目的 (第 2 章)

**怀疑** (1 项):
- 张三可能发现了自己的身份 (第 5 章)

**明确不知道** (2 项):
- 宝藏的具体位置
- 师父的遗言内容
```

## 如何使用

### 1. 初始化（自动）

系统会在章节生成开始时自动加载 `character_knowledge.md`：

```typescript
await this.knowledgeTracker.load();
```

如果文件不存在，会使用空状态。

### 2. 手动标记角色不知道的事实

在 `character_knowledge.md` 中的 **明确不知道** 部分添加：

```markdown
## 张三 (zhang-san)

**明确不知道** (3 项):
- 李四是卧底
- 师父的遗言内容
- 宝藏的真正用途
```

系统会在生成时严格检查，如果 AI 让张三提及这些内容，会被标记为 **critical** 违规。

### 3. 自动验证（每个 beat 生成后）

系统会自动验证每个 beat：

```typescript
const validation = await this.knowledgeValidator.validateBeat(
  prose,
  beat,
  chapterNumber,
  beatIndex
);

if (validation.action === "reject") {
  // 严重违规，拒绝这个 beat
} else if (validation.action === "warn") {
  // 警告但接受
}
```

### 4. 自动提取新知识

系统会从生成的文本中自动提取知识：

```typescript
// 检测模式：
// "张三知道李四是卧底" → 添加到张三的知识
// "张三怀疑赵六是黑手" → 添加到张三的怀疑
// "张三发现了密码" → 添加到张三的知识

this.knowledgeValidator.extractAndUpdateKnowledge(prose, beat, chapterNumber);
```

### 5. 查看验证结果

生成过程中会输出验证结果：

```
✓ 节拍 1 完成：180 字，累计 180 字
⚠️ 节拍 2 知识边界警告 (得分: 75)
  - 张三: 李四是卧底
❌ 节拍 3 知识边界严重违规，拒绝
  - 张三: 师父的遗言内容
✓ 节拍 4 知识验证通过 (1 个轻微问题)
```

## 验证规则

### 1. 明确不知道的事实（Critical）

如果角色提及了标记为"明确不知道"的事实，会被标记为 **critical** 违规：

```
❌ 严重违规
角色: 张三
违规事实: 李四是卧底
证据: "张三冷笑道：'李四，我早就知道你是卧底了。'"
建议: 张三不应该知道"李四是卧底"，请移除相关内容或改为推测语气
```

### 2. 将怀疑当作确定事实（Major）

如果角色用确定语气说出只是怀疑的事情，会被标记为 **major** 违规：

```
⚠️ 主要违规
角色: 张三
违规事实: 赵六是幕后黑手
证据: "张三肯定地说：'赵六就是幕后黑手。'"
建议: 张三只是怀疑"赵六是幕后黑手"，不应该用确定语气，建议使用"也许"、"可能"等词
```

### 3. 提及其他角色的私密信息（Warning）

如果角色提及了其他角色的私密信息，会给出警告：

```
⚠️ 警告
张三提及了李四的私密信息"组织的真正目的"，请确认是否合理
```

## 配置选项

在 `chapter-pipeline.ts` 中可以配置验证器：

```typescript
this.knowledgeValidator = createKnowledgeValidator(this.knowledgeTracker, {
  strictMode: false,  // 严格模式：任何违规都拒绝
  autoRetry: true,    // 自动重试
  maxRetries: 2,      // 最大重试次数
});
```

### strictMode: false（默认，推荐）

- **Critical 违规**：自动重试（最多 2 次），超过后拒绝
- **Major 违规**：2 个或以上自动重试，1 个警告但接受
- **Minor 违规**：警告但接受

### strictMode: true（严格）

- **Critical 违规**：直接拒绝
- **Major 违规**：自动重试，超过后拒绝
- **Minor 违规**：警告但接受

## 最佳实践

### 1. 在创建书籍时规划知识分布

在 `story_bible.md` 或 `volume_outline.md` 中规划：

```markdown
## 关键信息分布

- **李四是卧底**：只有李四自己知道，第 10 章被张三发现
- **师父的遗言**：只有师父和王五知道，第 15 章揭示
- **宝藏的真正用途**：无人知道，第 20 章揭晓
```

### 2. 手动维护 character_knowledge.md

在关键章节后手动更新：

```markdown
## 张三 (zhang-san)

**确定知道** (新增):
- 李四是卧底 (第 10 章) - 通过跟踪发现

**明确不知道** (移除):
- ~~李四的真实身份~~ (已在第 10 章得知)
```

### 3. 定期检查知识差异报告

使用 `generateKnowledgeDiffReport()` 查看角色知识差异：

```typescript
const report = this.knowledgeTracker.generateKnowledgeDiffReport([
  "zhang-san",
  "li-si",
  "zhao-liu"
]);
console.log(report);
```

### 4. 处理信息传递场景

当角色之间传递信息时，手动更新：

```markdown
## 第 12 章：张三告诉王五真相

**更新**:
- 王五现在知道"李四是卧底" (第 12 章) - 张三告知
```

## 常见问题

### Q: 系统会自动检测所有知识违规吗？

A: 系统会检测大部分明显的违规，但不是 100% 准确。建议：
- 手动标记关键的"明确不知道"事实
- 定期检查生成的内容
- 使用严格模式提高检测率

### Q: 如果 AI 用隐晦的方式暗示角色知道某事怎么办？

A: 系统会检测关键词匹配，但可能漏掉非常隐晦的暗示。建议：
- 在 `character_knowledge.md` 中详细描述禁止的内容
- 使用多个关键词变体

### Q: 系统会影响生成速度吗？

A: 影响很小：
- 验证是纯文本匹配，非常快
- 只在发现违规时才重试
- 平均每章增加 < 1 秒

### Q: 可以禁用知识追踪吗？

A: 可以，但不推荐。如果需要禁用：
- 删除 `story/character_knowledge.md`
- 系统会自动跳过验证

```markdown
## 设定
- 第 5 章：张三在密室发现"宝藏地图"
- 第 6 章：李四登场，不应该知道地图的事

## character_knowledge.md

### 张三 (zhang-san)
**确定知道**:
- 密室中有宝藏地图 (第 5 章)

### 李四 (li-si)
**明确不知道**:
- 密室中有宝藏地图
- 张三发现了地图

## 验证结果

"李四说：'我知道你找到了宝藏地图。'"

系统会标记为 **critical** 违规并拒绝。
```

### 场景 2：推测 vs 确定

```markdown
## 设定
- 第 8 章：张三怀疑赵六是叛徒
- 第 9 章：张三不应该用确定语气

## character_knowledge.md

### 张三 (zhang-san)
**怀疑**:
- 赵六可能是叛徒 (第 8 章)

## 验证结果

第 9 章生成时，如果 AI 写：
"张三肯定地说：'赵六就是叛徒。'"

系统会标记为 **major** 违规并建议重试。

正确的写法：
"张三皱眉道：'我怀疑赵六可能是叛徒。'"
```

## 技术细节

### 知识匹配算法

系统使用多层匹配：

1. **直接包含**：文本直接包含事实
2. **关键词匹配**：提取关键词，匹配 70% 以上
3. **确定性检测**：检测"知道"、"确定"、"肯定"等词

### 自动提取模式

```typescript
// 确定知识
/(.+?)(?:知道|得知|发现|意识到|明白|了解到)(?:了)?(.+?)[。！？]/g

// 怀疑/推测
/(.+?)(?:怀疑|猜测|推测|觉得|认为)(.+?)[。！？]/g
```

### 得分计算

```typescript
初始分数: 100
- Critical 违规: -30 分
- Major 违规: -15 分
- Minor 违规: -5 分

最终得分 < 60: 建议重试
最终得分 < 80: 警告
最终得分 >= 80: 接受
```

## 总结

知识追踪系统通过**自动验证 + 手动标记**的方式，有效防止 AI 让角色说出不应该知道的信息，保证剧情逻辑的一致性。

**关键点**：
- ✅ 自动验证每个 beat
- ✅ 自动提取新知识
- ✅ 提供修复建议
- ✅ 支持严格/宽松模式
- ✅ 最小性能影响

**使用建议**：
- 在 `character_knowledge.md` 中标记关键的"明确不知道"事实
- 定期检查生成内容
- 使用默认的宽松模式（平衡质量和速度）

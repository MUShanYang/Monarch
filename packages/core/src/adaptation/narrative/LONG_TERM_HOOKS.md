# 长期 Hook 使用指南

## 什么是长期 Hook

长期 Hook 是指贯穿全书、从开头到结尾才解决的剧情线索。例如：
- 主角的终极目标（复仇、寻找真相、拯救世界）
- 核心谜团（主角的身世、世界的秘密）
- 主要角色关系的发展（从敌对到盟友）
- 角色成长弧线（从弱小到强大）

## 为什么需要特殊处理

普通 Hook 优先级调整器会：
- 超过 10 章未解决 → 提升优先级
- 接近书籍结尾 → 标记为 critical
- 长时间未提及 → 要求立即推进

但长期 Hook 的特点是：
- **不应该过早解决** - 否则失去主线驱动力
- **需要定期提及** - 保持读者记忆和期待
- **分阶段推进** - 逐步揭示，而非一次性解决

## 如何标记长期 Hook

在 `pending_hooks.md` 中添加长期 Hook 时，使用以下格式：

```markdown
## 主角寻找失踪的师父

**状态**: open
**紧急度**: progressing
**类型**: mystery
**起源章节**: 1
**最后提及**: 5
**计划解决章节**: 95
**是否长期**: true

主角的师父在十年前神秘失踪，留下一块玉佩。主角踏上寻找师父的旅程。
```

关键字段：
- `计划解决章节`: 你计划在哪一章解决这个 hook（如第 95 章）
- `是否长期`: 设置为 `true` 表示这是长期 hook

## 长期 Hook 的优先级策略

系统会根据当前章节和计划解决章节的距离，自动调整优先级：

### 1. 远离解决阶段（距离 > 20 章）
- **优先级**: deferred
- **策略**: 偶尔提及即可，保持存在感
- **建议频率**: 每 10-15 章提及一次

### 2. 中期阶段（距离 10-20 章）
- **优先级**: normal
- **策略**: 开始铺垫，逐步揭示线索
- **建议频率**: 每 5-8 章推进一步

### 3. 接近解决（距离 3-10 章）
- **优先级**: urgent
- **策略**: 加速推进，准备高潮
- **建议频率**: 每 2-3 章推进

### 4. 即将解决（距离 ≤ 3 章）
- **优先级**: critical
- **策略**: 必须解决，进入高潮
- **建议频率**: 每章都要推进

## 停滞检测

即使是长期 Hook，也不能长时间不提及：

- **超过 10 章未提及** → 自动提升优先级，建议适当提及
- **超过 20 章未提及** → 标记为 critical，读者可能已经忘记

## 阶段性推进建议

系统会根据进度自动建议推进时机：

- **30%-50% 进度** → 建议推进一步（揭示部分线索）
- **60%-80% 进度** → 需要加速推进（揭示关键信息）
- **80%-100% 进度** → 进入解决阶段

## 示例：完整的长期 Hook 规划

假设你的小说计划写 100 章，主角寻找师父是核心主线：

```markdown
## 寻找失踪的师父

**计划解决章节**: 95
**是否长期**: true
**类型**: mystery

### 推进时间表

- 第 1 章：引入 hook（师父失踪，留下玉佩）
- 第 10 章：首次线索（玉佩发光，指向北方）
- 第 25 章：第二次线索（遇到师父的旧友，得知线索）
- 第 40 章：重大发现（发现师父的遗物）
- 第 55 章：转折点（发现师父可能还活着）
- 第 70 章：关键信息（得知师父被困的位置）
- 第 85 章：准备阶段（收集解救师父的资源）
- 第 95 章：高潮（找到并解救师父）
```

## 与普通 Hook 的对比

| 特性 | 普通 Hook | 长期 Hook |
|------|----------|----------|
| 解决时间 | 5-15 章内 | 贯穿全书 |
| 优先级策略 | 年龄越大越紧急 | 根据距离解决章节调整 |
| 提及频率 | 频繁 | 定期但不频繁 |
| 停滞容忍度 | 5 章 | 10-20 章 |
| 推进方式 | 快速解决 | 分阶段逐步推进 |

## 最佳实践

1. **明确计划解决章节** - 不要模糊，要有具体的章节数
2. **定期提及** - 即使不推进，也要让读者记得这条线索
3. **分阶段揭示** - 不要一次性揭示所有信息
4. **与其他 Hook 配合** - 长期 Hook 是主线，短期 Hook 是支线
5. **灵活调整** - 如果发现计划不合理，可以修改计划解决章节

## 在代码中使用

在 `chapter-pipeline.ts` 中，系统会自动识别长期 Hook：

```typescript
const hooks: HookMetadata[] = this.currentSnapshot.ledger.hooks.map(hook => ({
  id: hook.id,
  originChapter: hook.originChapter,
  lastReferencedChapter: hook.lastReferencedChapter,
  status: hook.status,
  urgency: hook.urgency,
  description: hook.description,
  type: hook.type,
  relatedCharacters: [],
  tags: [],
  plannedResolutionChapter: hook.plannedResolutionChapter, // 从 pending_hooks.md 读取
  isLongTerm: hook.isLongTerm, // 从 pending_hooks.md 读取
}));
```

系统会自动应用长期 Hook 的优先级策略，确保：
- 不会过早解决
- 定期保持存在感
- 在合适的时机推进
- 接近结尾时加速解决
